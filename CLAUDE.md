npm run build        # Prisma generate + React Router build

No test suite is configured.

## Architecture

**Stack:** React Router v7 (SSR), TypeScript, Tailwind CSS v4, Prisma (MongoDB), Vite.

**Routing:** File-based routes defined in [app/routes.ts](app/routes.ts). Each route file exports a `loader` (server data fetching) and `action` (mutations), plus a default React component. Route types are auto-generated by `react-router typegen` into `.react-router/types/`.

**Data layer:** [db.server.ts](db.server.ts) exports a singleton PrismaClient. Model files in [app/models/](app/models/) are named `*.server.ts` and contain typed query functions that call `db` directly — they are never imported on the client. MongoDB is used via Atlas; the Prisma provider is `mongodb`.

**Key models:**
- `Corrida` — running activities synced from the Strava API
- `despesas` — personal expenses with a `brassaco` flag (shared expense tracking)
- `pagamentosBrassaco` — payments against the shared (brassaco) balance
- `biblioteca` — books read, with optional `paginas` field (older records may lack timestamps)

**Strava sync flow:** [app/services/strava/](app/services/strava/) — `client.server.ts` calls the Strava API, `mapper.ts` transforms responses to DTOs, `repository.ts` wraps Prisma writes, `sync.server.ts` orchestrates incremental vs. full sync. The Corridas model re-exports the sync entry point.

**File uploads:** [app/models/pocketbase.server.ts](app/models/pocketbase.server.ts) uploads expense receipts to a self-hosted PocketBase instance and returns a public URL stored in `despesas.comprovante`.

**UI components:** Shadcn/ui components live in [app/components/ui/](app/components/ui/). Feature components are grouped by domain under [app/components/despesas/](app/components/despesas/), [app/components/corridas/](app/components/corridas/), etc. Icons are from `@hugeicons/react`.

**Layout:** [app/root.tsx](app/root.tsx) wraps all routes in a collapsible sidebar layout (`AppSidebar` + `SidebarInset`), except `/kiosk` which renders full-screen without the sidebar.

**Path alias:** `~` resolves to `app/` (configured via `vite-tsconfig-paths`).

## Environment Variables

Required in `.env`:
- `DATABASE_URL` — MongoDB Atlas connection string
- `CLIENT_ID`, `CLIENT_SECRET`, `REFRESH_TOKEN` — Strava OAuth credentials
- `POCKETBASE_URL`, `POCKETBASE_ADMIN_EMAIL`, `POCKETBASE_ADMIN_PASSWORD`, `POCKETBASE_COLLECTION`, `POCKETBASE_FIELD` — PocketBase file storage
- `MAPBOX_ACCESS_TOKEN` — Mapbox for route maps on the Corridas page

## MongoDB / Prisma Notes

The `despesas` model uses `$runCommandRaw` for filtered queries (date range filtering) because Prisma's MongoDB driver doesn't support all MongoDB query operators natively. New queries that filter `despesas` by date should follow the same `montarFiltroData` pattern in [app/models/despesas.server.ts](app/models/despesas.server.ts).

Run `npx prisma generate` after any schema change (also runs automatically in `npm run build`).
